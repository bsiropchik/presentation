SVG

Scalable Vector Graphics (SVG)
is a Web graphics language.

SVG is an application of XML that makes it possible to represent graphic information in a compact form.

SVG ADVANTAGES

  • Scalable
  • Resolution Independent
  • Can be animated
  • Interactive & styleable
  • possibility to use js/css
  • file is smaller than jpeg/png
It is good to use in adaptive design,
because we can scale svg up and down
without loss of quality
However, svg is not very suitable
for some highly detailed images

Creating SVG

Since svg is XML-based format,
it may be created in any text editor.

However it is more convenient to use
vector graphic editors (Adobe Illustrator, Xara)
or any other which allows to export to *.svg

Text Editor

  • Create svg tag
  • Add necessary attributes
  • Create element inside of svg tag

Graphic Editor

When working in visual editor,
all you have to do is create image
and save it as *.svg file

SVG Text

The <text> element is used to define a text.
It's possible to apply a gradient, pattern, clipping path, mask, or filter to <text>

Groups

The <g> is a container used to group other SVG elements. Transformations applied to the <g> element are performed on all of its child elements, and any of its attributes are inherited by its child elements.

<defs>

This tag is used to store parts that will be used at a later time. Objects created inside a <defs> are not rendered directly.

<use>

You can use objects stored in <defs> with this tag. You need to indicate svg id or path to the source file.

Filters

Filters are used to alter image.

  • <feBlend> - filter for combining images
  • <feColorMatrix> - filter for color transformations
  • <feOffset> - filter for drop shadows
  • <feSpotLight> - filter for lighting
  • <feGaussianBlur> - blur filter
  • <feMerge> - filter to control other filters
  • ...and many others

Gradients

Gradient properties are set within a <linearGradient> element. It should be stored within the <defs> tag.
Linear gradients can be defined as horizontal, vertical or angular gradients

It may be applied to any difficult shape, which use fill property

Patterns

The SVG <pattern> element allows to define patterns inside of SVG and use those patterns as a fill.
The basic process for patterns goes something like:
  • Define a <pattern> inside of the SVG
  • Define the shapes inside of the pattern
  • Use the shapes
  • Create a new shape and fill it with the pattern

Transformations

The transform attribute defines a list of transform definitions that are applied to an element and the element's children.

Animation

The animate tag is put inside a shape element and defines how an attribute of an element changes over the animation.
<animateMotion> tag allows to set motion along the selected path
<animateTransform> changes a transformation attribute on a target element, allowing to control translation, scaling, rotation and/or skewing
SVG animation allows interactions, using mouse events such as click, mouseIn, mouseOut, mouseOver etc.

CSS animations also may be applied,
it works mostly the same way as with html elements
